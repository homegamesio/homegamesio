High Level Overview

Homegames is a game platform allowing people to play games built by the Homegames community. Everything we build is free and open source with a GNU GPL v3.0 license, with the exception of our game engine squish which has an MIT license). Games are JavaScript components that consist of a state that can be translated to a rendered state on a Homegames client using squish. Using the Homegames platform, users can sign up for an account and publish games of their own to share with friends or publish for the world to play.

Squish (our game engine) is a common dependency between Homegames core and Homegames web. It's basically a component that translates a game from "state stored in memory on the server" to "rendered image on the screen". "State stored in memory on the server" is Homegames core. "Rendered image on the screen" is our official Homegames web client or any other custom client.

As of July 2022, we don't have very thorough documentation for much of the process of game development or publishing. We have documentation for Squishjs (the Homegames game engine) available here which at this point is probably the best documentation we have. More documentation is coming soon, specifically around building games and the different features we support. We're planning on having a full guide on building and publishing your own game by July 30, 2022.

A Bit More Technical

PDF Version

A homegames client is any device with a web browser.

To connect to a Homegames client server (Homegames web server), a client device makes a GET request to homegames.link, which will redirect the client to the local IP (eg. 192.12.1.2) of the Homegames client server

Once the device downloads the web client from the client server, the client opens a websocket connection to the Homegames core server, which is responsible for running games available on the machine (in other words, games stored on disk). The core server sends serialized representations of game state over the websocket connection to the client, which then renders the game to the screen. This state serialization is done by our game engine, squish.

The Homegames core server maintains a websocket connection to homegames.link while running, which lets homegames.link keep track of active Homegames servers and provide mappings to clients when they go to homegames.link in the browser. This is completely optional and can be turned off via config.json (LINK_ENABLED value)

The Homegames core server downloads games and assets from S3 via Landlord, a service that keeps track of Homegames data - mostly in DynamoDB. The core server keeps this data on disk and provides it to clients when they need it (eg. a sound asset that plays when the player taps the screen). Landlord is also responsible for handling search queries made by the Homegames dashboard, as well as providing info about how the core server can access the corresponding game code.

Both homegames-core and homegames-web depend on multiple versions of squishjs. They both have config files called "squish maps" that point from a verison key (eg. '0750') to the corresponding version of squish from NPM.

This allows games to depend on specific versions by version key which is used by Homegames core at runtime to provide the correct environment variable for SQUISH_PATH based on the version specified when submitting a game for publishing. It also allows us to maintain backwards compatibilty with games that depend on older/deprecated versions of the engine as we release new versions.
